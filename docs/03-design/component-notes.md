# Component Notes

This repo is markdown-first, but it still has recurring structural components.

## Core recurring components

### Product seed
Used to capture the first usable version of the product idea.

### Product brief
Used to state what the product is, why it exists, who it is for, and what the MVP is.

### Functional spec
Used to define the product behavior at a system level.

### Current sprint
Used to constrain implementation to the current planned increment.

### Change request
Used to make requirement changes explicit and traceable.

### Done definition
Used to prevent fake completion.

## Structural rules
- Prefer updating these recurring components over inventing one-off docs.
- Keep each file focused on one role.
- Avoid duplicate planning across multiple files.
