# Copy Guidelines

## Tone
Direct, operational, and unpretentious.

## Writing rules
- Prefer direct labels.
- Prefer short helper text.
- Avoid vague marketing filler unless intentionally writing launch copy.
- Avoid pretending certainty where the repo only provides structure.
- Explain the role of a file or step in plain language.

## Repo-specific guidance
- README should sound useful, not inflated.
- SKILL.md should sound precise, not mystical.
- AGENTS.md should sound strict enough to prevent drift.
