# Target User

## Primary user
A solo builder or small-team operator using GPT and/or Codex to build products inside a repository.

## User goals
- start from a vague idea without chaos
- keep the project coherent over time
- update requirements without losing the thread
- know what is actually done vs. not done

## User frustrations
- AI jumping into code too early
- stale specs
- task drift
- sessions losing context
- fake “done” claims

## What success looks like for the user
The repo itself keeps product work organized enough that future sessions can continue productively.
