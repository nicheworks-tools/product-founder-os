# Success Criteria

## Product success
- a new user can understand what the repo does from the README
- the repo structure clearly supports product work
- the repo does not feel like random filler

## MVP success
- README / SKILL / AGENTS are coherent
- core docs are populated enough to demonstrate the operating model
- helper scripts work
- examples are useful enough to show intended usage

## Launch success
- the repo can be uploaded publicly without obvious placeholder weakness
- the main workflow is understandable
- the repo gives a believable starting point for real projects
