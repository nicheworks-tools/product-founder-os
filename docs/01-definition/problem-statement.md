# Problem Statement

## Core problem
AI can generate docs and code quickly, but product work still falls apart when definition, planning, execution, and change handling are disconnected.

## Why current options are insufficient
Many setups are either:
- prompt collections with no durable state
- code-first repos with weak planning
- docs-heavy repos with no execution rhythm

## Why this product is worth building now
As GPT and Codex become more repo-connected, a stronger operating structure becomes more useful than isolated skills.
