# Workflow

## Phase 1 — Discover
Ask what the user wants to build and capture the initial product seed.

Primary files:
- `docs/00-intake/project-seed.md`
- `docs/00-intake/assumptions.md`

## Phase 2 — Define
Clarify the product, user, problem, scope, non-goals, and intended behavior.

Primary files:
- `docs/01-definition/*`
- `docs/02-spec/*`
- `docs/03-design/*`

## Phase 3 — Plan
Convert the definition into milestones, backlog, and a current sprint.

Primary files:
- `docs/04-planning/roadmap.md`
- `docs/04-planning/milestone-plan.md`
- `docs/04-planning/task-backlog.md`
- `docs/04-planning/current-sprint.md`

## Phase 4 — Build
Implement against the current sprint and keep worklog/task status current.

Primary files:
- implementation files
- `docs/05-execution/worklog.md`

## Phase 5 — Adjust
When the user changes direction, update the source-of-truth docs first.

Primary files:
- `docs/05-execution/change-requests.md`
- affected spec/planning docs

## Phase 6 — Ship
Evaluate whether the product is actually ready for the intended milestone or release.

Primary files:
- `docs/06-quality/*`
- `docs/07-handoff/*`
