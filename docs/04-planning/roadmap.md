# Roadmap

## Phase 1 — Discover
- Capture what the user wants to build.
- Remove ambiguity.

## Phase 2 — Define
- Write brief, scope, success criteria, and spec.

## Phase 3 — Plan
- Build milestones, backlog, and current sprint.

## Phase 4 — Build
- Implement against the current sprint.

## Phase 5 — Adjust
- Absorb changes without losing coherence.

## Phase 6 — Ship
- Confirm done state, handoff, and release checklist.
