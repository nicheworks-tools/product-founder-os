# Project Seed

## Current idea
Build a repository operating system that helps GPT and/or Codex turn a vague product idea into a structured product workflow and then keep moving that product toward completion.

## Who it is for
- solo builders
- small teams
- people using GPT and/or Codex against a repo
- projects that need product definition before implementation

## Problem
AI-assisted product work often starts too early in code, loses scope coherence, drifts across sessions, and ends up with stale docs and fake completion signals.

## MVP target
A public-quality repo that gives a clear workflow for:
- discovery
- definition
- planning
- execution
- adjustment
- handoff / completion

## Constraints
- must work as a repo-first workflow
- must be useful with GPT and/or Codex
- should stay compact and not become doc spam
- should be understandable as a public repo

## Open questions
- how strong the examples should become in v1
- whether a starter bootstrap script should be expanded later
