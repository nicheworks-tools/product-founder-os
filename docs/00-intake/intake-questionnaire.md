# Intake Questionnaire

Use these questions when the user has not clearly defined the product yet.

## Product basics
- What do you want to build?
- Who is it for?
- What problem does it solve?
- What would make the product feel useful immediately?

## MVP
- What absolutely must exist in v1?
- What can wait until later?
- What should explicitly stay out of scope for now?

## Constraints
- Is this public, internal, or experimental?
- Web app, static site, mobile app, script, API, or something else?
- Any tech constraints?
- Any time, budget, or hosting constraints?

## UX / design
- Speed-first, utility-first, brand-first, or visual polish-first?
- Desktop-first, mobile-first, or both?
- Any reference products or design directions?

## Success
- What would count as "done" for the next milestone?
- What would count as launch-ready?
