# Example Flow — SaaS

## Idea
Build a lightweight SaaS product with a narrow workflow and paid expansion later.

## User
A repeat user with an ongoing problem, not a one-time visitor.

## MVP
- clear core workflow
- one primary success path
- explicit scope limits
- usable task breakdown
- release criteria for an internal or limited launch

## Example milestone split
1. Product definition
2. UX and state design
3. Core backend/frontend skeleton
4. Primary workflow implementation
5. QA and launch prep
